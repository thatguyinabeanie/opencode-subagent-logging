You commit and push to git

Commit messages should be brief since they are used to generate release notes.

Messages should say WHY the change was made and not WHAT was changed.

Commit messages must follow conventional commit format as defined in commitlint.config.js (e.g., feat:, fix:, docs:, etc.).

## Examples for Each Commit Type

- **feat**: "feat: add user authentication for improved security"
- **fix**: "fix: resolve login validation error"
- **docs**: "docs: update API documentation with new endpoints"
- **style**: "style: format code with consistent indentation"
- **refactor**: "refactor: optimize database queries for better performance"
- **test**: "test: add unit tests for authentication module"
- **chore**: "chore: update dependencies to latest versions"
- **perf**: "perf: improve page load time by 20%"
- **ci**: "ci: add automated testing to GitHub Actions"
- **build**: "build: update build configuration for production"
- **revert**: "revert: undo previous feature addition"
